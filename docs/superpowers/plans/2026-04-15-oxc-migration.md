# OXC Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace ESLint + Prettier with `oxlint` + `oxfmt` in a single PR, keeping Stylelint for CSS.

**Architecture:** Two sequential tasks — first wire in the new tools (add packages, create configs, update scripts), then tear out the old ones (remove packages, delete config files, reformat the codebase). This order ensures the repo is never in a broken state between commits.

**Tech Stack:** Yarn 4 (Yarn Berry), `oxlint` (linter), `oxfmt` (formatter), `stylelint` (CSS, unchanged)

## Task 2: Remove old tooling, reformat codebase, delete old configs

**Files:**

- Modify: `package.json` — remove 7 devDependencies
- Delete: `eslint.config.mjs`
- Delete: `.prettierrc.json`
- Delete: `.prettierignore`

- [ ] **Step 1: Remove old packages**

```bash
yarn remove eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-config-prettier eslint-plugin-storybook @eslint/json prettier
```

Expected: those 7 entries disappear from `devDependencies` in `package.json` and `yarn.lock` is updated. All other deps are unchanged.

- [ ] **Step 2: Delete old config files**

```bash
rm eslint.config.mjs .prettierrc.json .prettierignore
```

- [ ] **Step 3: Reformat the codebase with oxfmt**

```bash
yarn format
```

Expected: oxfmt rewrites any files whose formatting differs from its output. Since the config options are identical to Prettier's, most files will be unchanged. A small number may have minor whitespace differences. This is expected — commit them as part of the migration.

- [ ] **Step 4: Verify lint passes**

```bash
yarn lint
```

Expected: exits 0, no errors.

- [ ] **Step 5: Verify format check passes**

```bash
yarn format:check
```

Expected: exits 0 — all files are now formatted by oxfmt.

- [ ] **Step 6: Verify CSS lint still passes**

```bash
yarn lint:css
```

Expected: stylelint runs and exits 0 (unchanged tool, unchanged config).

- [ ] **Step 7: Verify tests still pass**

```bash
yarn test
```

Expected: all 37 tests pass across 3 suites.

- [ ] **Step 8: Commit**

```bash
git add -A
git commit -m "feat: remove eslint and prettier, complete oxc migration"
```

The `-A` here is intentional — it stages the deleted config files, the reformatted source files, and the updated `package.json`/`yarn.lock` all at once.
