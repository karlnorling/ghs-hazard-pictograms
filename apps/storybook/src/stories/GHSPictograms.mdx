# GHS Hazard Pictograms

The **Globally Harmonized System (GHS) of Classification and Labelling of Chemicals** defines
a set of standardised pictograms used worldwide on chemical packaging and safety data sheets.

This package exposes all 9 core GHS pictograms (plus transport hazard classes) in three
complementary formats:

| Format           | Import path                            | Best for                                                   |
| ---------------- | -------------------------------------- | ---------------------------------------------------------- |
| React components | `ghs-hazard-pictograms/react`          | React/JSX projects — inline SVG, full a11y, tree-shakeable |
| SVG sprite       | `ghs-hazard-pictograms/sprite.svg`     | Any HTML — single HTTP request, reference with `<use>`     |
| CSS sprite       | `ghs-hazard-pictograms/css/sprite.css` | Class-name driven — easy to apply in HTML or any framework |

### Props reference — `PictogramProps`

| Prop          | Type                  | Default                             | Description                                           |
| ------------- | --------------------- | ----------------------------------- | ----------------------------------------------------- |
| `title`       | `string`              | Pictogram name (e.g. `"Explosive"`) | Accessible title injected as `<title>` inside the SVG |
| `description` | `string`              | Wikipedia hazard description        | Accessible description injected as `<desc>`           |
| `width`       | `number \| string`    | SVG intrinsic width                 | Width applied to the `<svg>` element                  |
| `height`      | `number \| string`    | SVG intrinsic height                | Height applied to the `<svg>` element                 |
| `className`   | `string`              | —                                   | CSS class forwarded to the wrapping `<span>`          |
| `style`       | `React.CSSProperties` | —                                   | Inline styles forwarded to the wrapping `<span>`      |
| `aria-label`  | `string`              | —                                   | Overrides `aria-labelledby` with a direct label       |

### Accessibility

Every component renders with `role="img"`, `aria-labelledby` pointing to an injected
`<title>` and `<desc>` element. Screen readers announce the pictogram name and hazard
description without any extra work.

To override the defaults:

```tsx
<Ghs06Toxic
  title="Danger: Toxic substance"
  description="Acutely toxic — wear PPE before handling."
  width={120}
/>
```

## SVG Sprite

A single `sprite.svg` file contains all pictograms as `<symbol>` elements, enabling efficient
reuse with a single HTTP request.

<Canvas of={SVGSpriteStories.AllSymbols} />

### External file reference

```html
<svg width="120" height="120" role="img" aria-label="GHS01 Explosive">
  <title>GHS01 Explosive</title>
  <use
    href="/assets/sprites/sprite.svg#physical-hazards-pictograms-ghs01-explosive-ghs-pictogram-explos"
  />
</svg>
```

### Inline sprite (reference many times from one embed)

```html
<!-- Embed once at the top of your document -->
<div aria-hidden="true" style="position:absolute;width:0;height:0;overflow:hidden">
  <!-- contents of sprite.svg go here -->
</div>

<!-- Reference anywhere -->
<svg width="120" height="120" role="img" aria-label="GHS01 Explosive">
  <title>GHS01 Explosive</title>
  <use href="#physical-hazards-pictograms-ghs01-explosive-ghs-pictogram-explos" />
</svg>
```

### React import

```tsx
import spriteUrl from 'ghs-hazard-pictograms/sprite.svg';

<svg width={120} height={120} role="img" aria-label="GHS01 Explosive">
  <title>GHS01 Explosive</title>
  <use href={`${spriteUrl}#physical-hazards-pictograms-ghs01-explosive-ghs-pictogram-explos`} />
```

## CSS Sprite

Import `ghs-hazard-pictograms/css/sprite.css` to get a set of `.ghs-*` utility classes.
Apply a class to any block element and size it with `width` / `height`.

<Canvas of={CSSSpriteStories.AllPictograms} />

### Setup

```js
// In your entry point (JS/TS/CSS):
import 'ghs-hazard-pictograms/css/sprite.css';
```

### Usage

```html
<span
  class="ghs-physical_hazards_pictograms_ghs01_explosive_ghs-pictogram-explos"
  style="display:inline-block;width:120px;height:120px"
  role="img"
  aria-label="GHS01 Explosive"
  title="GHS01 Explosive"
></span>
```

### CSS helper (optional)

```ts
import { getCssClassName } from 'ghs-hazard-pictograms/css';

const cls = getCssClassName('ghs01-explosive');
// → 'ghs-physical_hazards_pictograms_ghs01_explosive_ghs-pictogram-explos'

element.className = cls;
```

### Generated CSS pattern

Each pictogram generates one CSS rule:

```css
.ghs-physical_hazards_pictograms_ghs01_explosive_ghs-pictogram-explos {
  background-image: url('../physical_hazards_pictograms/ghs01_explosive/GHS-pictogram-explos.svg');
  background-size: contain;
  background-repeat: no-repeat;
  background-position: center;
}
```

## Assets

All pictograms are also available as pre-built image files at multiple resolutions:

```
assets/
  physical_hazards_pictograms/
    ghs01_explosive/
      GHS-pictogram-explos.svg          ← original SVG
      GHS-pictogram-explos_240x240.png  ← raster variants
      GHS-pictogram-explos_512x512.png
      GHS-pictogram-explos_768x768.png
      GHS-pictogram-explos_1024x1024.png
      GHS-pictogram-explos_2048x2048.png
      GHS-pictogram-explos_240x240.jpg
      GHS-pictogram-explos_240x240.webp
      ...
```

Import individual assets from the `assets/` path:

```ts
import explos from 'ghs-hazard-pictograms/assets/physical_hazards_pictograms/ghs01_explosive/GHS-pictogram-explos.svg';
```

Or use the `assets` map from the programmatic API:

```ts
import { getPictogram } from 'ghs-hazard-pictograms';

const p = getPictogram('ghs01-explosive');
console.log(p?.assets.png[512]); // path to 512×512 PNG
```

## Programmatic API

```ts
import {
  getAllPictograms,
  getGHSPictograms,
  getPictogram,
  getPictogramsByCategory,
} from 'ghs-hazard-pictograms';

// All 24 pictograms (GHS + transport)
const all = getAllPictograms();

// Only the 9 core GHS01–09 pictograms
const ghs = getGHSPictograms();

// Look up by ID
const explosive = getPictogram('ghs01-explosive');
// { id, code, name, category, description, svg, assets }

// Filter by category
const physical = getPictogramsByCategory('physical_hazards');
```

### Categories

| Value                         | Description                                                              |
| ----------------------------- | ------------------------------------------------------------------------ |
| `physical_hazards`            | GHS01–GHS05 (explosive, flammable, oxidising, compressed gas, corrosive) |
| `health_hazards`              | GHS06–GHS08 (toxic, health hazard, serious health hazard)                |
| `physical_and_health_hazards` | GHS05 Corrosive (appears in both)                                        |
| `environmental_hazards`       | GHS09 (hazardous to environment)                                         |
| `transport`                   | UN transport hazard classes (divisions 1.1–9)                            |
